## 🚀 集成步骤

### 1. 添加引用

在 `GPGems.Visualization.csproj` 中确保引用：
```xml
<ProjectReference Include="..\GPGems.AI\GPGems.AI.csproj" />
<ProjectReference Include="..\GPGems.Core\GPGems.Core.csproj" />
```

### 2. 在主窗口添加入口

在 `MainWindow.xaml` 的 TabControl 中添加：

```xml
<TabItem Header="🏰 经营演示">
    <local:ManorDemoManager />
</TabItem>
```

### 3. 编译运行

```bash
dotnet run --project GPGems.Visualization
```

切换到「经营演示」标签页即可使用。

## 🎯 各场景算法与颜色说明

### 🚶 场景1：游客人流
**算法：流场寻路 + ORCA避障**

| 颜色 | 状态 |
|------|------|
| 🔴 红色 | 前往景点A |
| 🔵 青色 | 前往景点B |
| 🟡 黄色 | 前往出口 |

### 🐟 场景2：动物群体
**算法：Boids三种参数对比**

| 颜色 | 动物 | 参数特点 |
|------|------|---------|
| 🔵 蓝色小 | 鱼群 | 高凝聚、高同步 |
| 🟤 棕色大 | 牛羊 | 低凝聚、慢速度、漫游 |
| 🟣 粉色小 | 蝴蝶 | 低同步、快转向、飘忽 |

### 🧑‍💼 场景3：员工任务
**算法：A*寻路 + FSM状态机**

| 颜色 | 状态 |
|------|------|
| ⚪ 灰色 | 空闲等待分配 |
| 🟡 黄色 | 移动中（显示路径 |
| 🟢 绿色 | 工作中 |

| 任务点颜色 | 类型 |
|-----------|------|
| 🟢 绿色 | 收菜 Harvest |
| 🟡 金色 | 喂食 Feed |
| 🔵 蓝色 | 服务 Serve |

### 🚨 场景4：紧急疏散
**算法：社会力模型**

| 颜色 | 含义 |
|------|------|
| 浅灰 → 红 | 距离出口越近，颜色越红 |
| 🟢 绿色区域 | 出口通道 |
| 🟩 亮绿区域 | 已疏散安全区 |

## 🔧 扩展开发

1. 实现 `IDemoScene` 接口：
```csharp
public class MyNewScene : IDemoScene
{
    public void Reset(int count, float speed) { ... }
    public void Update(float deltaTime) { ... }
    public void RenderBackground(Canvas c, List<Shape> cache) { ... }
    public void RenderAgents(Canvas c, List<Shape> cache) { ... }

```

2. 在 `ManorDemoManager.xaml.cs` 中：
   - 添加场景成员变量
   - 在 `InitializeScenes()` 中初始化
   - 在 `DemoSelector_SelectionChanged` 添加分支

## 💡 性能优化提示

- > 100单位时，性能瓶颈是渲染不是算法
- 超过200单位时，建议关闭WPF反锯齿
- 超过500单位时，考虑使用DirectX渲染（WPFD3DImage）

```csharp
// 关闭反锯齿（可选，提升性能
RenderOptions.SetEdgeMode(RenderCanvas, EdgeMode.Aliased);
```

## 📋 已知问题

1. 场景切换时没有清除上一个场景的Timer
   - 修复：在 `DemoSelector_SelectionChanged` 中先停止Timer再重置

2. 大数量（>300）时帧率下降
   - 原因：WPF每个Ellipse都是独立Visual
   - 优化：使用DrawingVisual批量渲染
