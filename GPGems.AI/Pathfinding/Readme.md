## 基础搜索算法

### BFS 广度优先搜索

**核心思想**：按层级逐层扩展，像水波一样扩散出去。

```
      ★ → ● → ● → ●
      ↓   ↓   ↓   ↓
      ● → ● → ● → ●
      ↓   ↓   ↓   ↓
      ● → ● → ● → ◎
```

**实现细节**：
- **数据结构**：`Queue<T>` 先进先出
- **时间复杂度**：O(V + E)
- **空间复杂度**：O(V)

**特点**：
- ✅ 在单位权重网格中保证找到最短路径
- ✅ 实现简单，直观易懂
- ❌ 搜索范围大，效率低
- ❌ 不支持非单位权重

**适用场景**：
- 简单迷宫游戏
- 状态空间搜索
- 最短路径步数计数

---

### DFS 深度优先搜索

**核心思想**：一条路走到黑，碰壁才回头。

```
      ★ → → → →
                ↓
      ← ← ← ← ←
      ↓
      → → → → ◎
```

**实现细节**：
- **数据结构**：`Stack<T>` 后进先出
- **时间复杂度**：O(V + E)
- **空间复杂度**：O(V)

**特点**：
- ✅ 内存占用小（栈深度）
- ✅ 实现极其简单
- ❌ 路径可能非常长
- ❌ 不保证最短路径
- ❌ 可能陷入死胡同

**适用场景**：
- 教学演示
- 状态空间遍历
- 记忆化搜索

---

### Dijkstra 算法

**核心思想**：始终扩展当前代价最小的节点，动态规划思想。

```
      ★(0) → ●(1) → ●(2)
        ↘     ↓     ↙
          ●(1.4) → ●(2.8) → ◎
```

**实现细节**：
- **数据结构**：优先队列 `PriorityQueue`
- **时间复杂度**：O((V + E) log V)
- **核心公式**：`g(neighbor) = g(current) + cost(current, neighbor)`

**特点**：
- ✅ 保证找到最短路径（全局最优）
- ✅ 支持任意非负权重
- ❌ 搜索方向盲目，向所有方向扩展
- ❌ 对于长距离寻路效率很低

**适用场景**：
- 路由协议（OSPF）
- 交通导航
- 加权图最短路径

---

### Best-First 贪心搜索

**核心思想**：只看启发式，始终向最接近目标的方向前进。

```
      ★
        ↘
          ●
            ↘
              ● → ◎
```

**实现细节**：
- **数据结构**：优先队列
- **排序键**：仅启发式距离 `h(n)`
- **时间复杂度**：O((V + E) log V)

**特点**：
- ✅ 速度极快，搜索范围小
- ✅ 很少绕路
- ❌ 不保证最短路径
- ❌ 可能陷入局部最优

**适用场景**：
- 快速原型验证
- 对路径质量要求不高的场景
- 实时性要求高的游戏

## A* 算法详解

### 核心原理

A* 是 Dijkstra 和 Best-First 的完美结合：

**f(n) = g(n) + h(n)**

- **g(n)**：从起点到当前节点的实际代价
- **h(n)**：当前节点到终点的启发式估计代价
- **f(n)**：总代价估计

**可采纳性**：当启发式不高估实际代价时，A* 保证找到最短路径

---

### 启发式函数

#### 1. 曼哈顿距离（Manhattan）
```csharp
h = |x1 - x2| + |y1 - y2|
```
- ✅ 4方向网格的完美可采纳启发式
- ❌ 8方向网格会高估

#### 2. 切比雪夫距离（Chebyshev）
```csharp
h = max(|x1 - x2|, |y1 - y2|)
```
- ✅ 对角线移动代价 = 直线移动代价时使用
- ❌ 代价不同时会高估

#### 3. Octile 距离（8方向网格最优）
```csharp
int dx = Abs(x1 - x2);
int dy = Abs(y1 - y2);
h = min(dx, dy) * 1.41421356 + Abs(dx - dy);
```
- ✅ 8方向网格的完美可采纳启发式
- ✅ 对角线代价 √2，直线代价 1
- ✅ 当前实现使用

#### 4. 欧几里得距离
```csharp
h = sqrt(dx² + dy²)
```
- ✅ 任意方向移动时使用
- ❌ 计算开销大（浮点数运算）

---

### A* 优化手段

#### ✅ 优化 1：使用 .NET 内置 PriorityQueue

**问题**：原来使用 `SortedSet` 需要自定义比较器，性能较低

**优化后**：
```csharp
// 优化前
var openSet = new SortedSet<(double f, int x, int y)>();

// 优化后
var openSet = new PriorityQueue<GridNode, double>();
openSet.Enqueue(node, node.FCost);
```

**性能提升**：
- 插入：O(log n) → 相同复杂度但常数更小
- 提取最小值：O(log n) → 原生支持
- 内存开销减少约 30%

---

#### ✅ 优化 2：Jump Point Search (JPS)

**核心思想**：跳过对称路径上的大部分节点，只在"跳点"处展开

```
普通 A*:  ● → ● → ● → ● → ●
JPS:      ● → → → → → → → ●
```

**优化效果**：
- 速度提升 10-100 倍
- 保持最优性
- 开放集大小减少 90%+

**关键规则**：
1. 直线移动：跳过强制邻居
2. 对角线移动：只在有强制邻居时停止
3. 跳点检测

---

#### ✅ 优化 3：Theta* 任意角度寻路

**问题**：网格路径有锯齿效应

```
网格路径:    ～～
            ～
         ～
      ～

Theta*:  ────────
```

**优化方法**：
- 在重建路径时检查视线
- 父节点不必是相邻节点
- 路径平滑自然

---

#### ✅ 优化 4：层级 A* (HPA*)

**核心思想**：
1. 将大图分块（cluster）
2. 先在块级粗粒度寻路
3. 再在块内细粒度寻路

**适用**：
- 大型开放世界游戏
- 万级节点以上的超大地图

---

#### ✅ 优化 5：Early Exit 提前终止

```csharp
// 传统：找到目标立即返回
if (current == goal)
    return ReconstructPath(start, goal);
```

**优化**：
- 不必等待所有可能路径都检查
- 找到第一个就是最优（启发式可采纳前提下）

---

#### ✅ 优化 6：邻居排序

```csharp
// 按启发式距离排序邻居，优先扩展更有希望的方向
var neighbors = map.GetNeighbors(current)
    .OrderBy(n => OctileDistance(n, goal))
    .ToList();
```

**效果**：
- 优先队列更快找到目标
- 无用节点展开更少
- 实际运行时间减少

---

#### ✅ 优化 7：路径平滑

```
原始路径:   ● → ●
              ↘
                ● → ●
                    ↘
                      ●

平滑后:     ● ──────────── ●
```

**算法**：
1. 从起点开始，尝试连接最远的可见节点
2. 如果不可见，回退一步
3. 重复直到终点

---

#### ✅ 优化 8：动态权重 A*

```csharp
// 动态调整启发式权重
double weight = IsFarFromGoal ? 1.5 : 1.0;
f = g + weight * h;
```

**效果**：
- 远距离时更贪心，速度更快
- 接近目标时恢复最优性

## 双向 A* 算法

### 核心原理

同时从起点和终点两个方向搜索，在中间相遇：

```
正向:  ★ → ● → ● ← ● ← ◎
                ↓
                相遇点
```

### 实现细节

**数据结构**：
- 两个优先队列：`forwardOpen` + `backwardOpen`
- 两个闭集：`forwardClosed` + `backwardClosed`
- 两个 G 值字典

### 相遇检测

```csharp
// 当前节点在对方闭集中时，计算总代价
if (backwardClosed.Contains(current))
{
    double totalCost = forwardG[current] + backwardG[current];
    if (totalCost < bestCost)
    {
        bestCost = totalCost;
        meetingPoint = current;
    }
}
```

### 路径拼接

```csharp
// 前向路径: start → ... → meeting
// 反向路径: meeting → ... → goal
// 合并:     start → ... → meeting → ... → goal
```

### 性能优势

| 距离 | 单向 A* | 双向 A* | 提升 |
|------|---------|---------|------|
| 10 步 | 100 节点 | ~30 节点 | 3.3x |
| 100 步 | 10,000 节点 | ~300 节点 | 33x |
| 1000 步 | 1,000,000 节点 | ~3,000 节点 | 333x |

**搜索区域**：O(r²) → O(2 × (r/2)²) = O(r²/2)
**实际提升**：长距离 50-70% 速度提升

## 总结

### 选择建议

| 需求 | 推荐算法 |
|------|---------|
| 必须最短路径，距离短 | A* |
| 必须最短路径，距离长 | 双向 A* |
| 只要快就行，路径无所谓 | Best-First |
| 教学演示，想看暴力搜索 | BFS / DFS |
| 动态权重地图 | Dijkstra / 加权 A* |
| 超大开放世界 | HPA* + JPS |

### 关键洞察

1. **启发式质量决定一切**：好的启发式 = 快 + 准
2. 双向搜索在长距离场景下收益巨大
3. 数据结构选择影响性能 30%+
4. 后处理平滑让路径看起来更自然
5. 没有免费午餐：速度 ↔ 质量的权衡永恒
